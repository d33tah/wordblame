'''Nmap''' (''Network Mapper'') is a security [[Network scanner|scanner]] originally written by [[Gordon Lyon]] (also known by his pseudonym ''Fyodor Vaskovich''){{cite news|title=Matrix mixes life and hacking|url=http://news.bbc.co.uk/1/hi/technology/3039329.stm|publisher=''[[BBC News]]''|accessdate=2009-01-14 | date=2003-05-19}} used to discover [[Host (network)|hosts]] and [[Web service|services]] on a [[computer network]], thus creating a "map" of the network. To accomplish its goal, Nmap sends specially crafted [[Network packet|packets]] to the target host and then analyzes the responses.

The software provides a number of features for probing computer networks, including host discovery and service and [[operating system]] detection. These features are extensible by scripts that provide more advanced service detection,name="Nmap Scripting Engine">[https://nmap.org/book/nse.html#nse-intro Nmap Scripting Engine]. Nmap.org. Retrieved on 2013-02-01. vulnerability detection,name="Nmap Scripting Engine"/> and other features. Nmap is also capable of adapting to network conditions including [[Network latency|latency]] and [[network congestion|congestion]] during a scan. Nmap is under development and refinement by its user [[community]].

Nmap was originally a [[Linux]]-only utility,[https://nmap.org/book/history-future.html The History and Future of Nmap]. Nmap.org. Retrieved on 2013-02-01. but it was ported to [[Microsoft Windows|Windows]], [[Solaris (operating system)|Solaris]], [[HP-UX]], [[BSD]] variants (including [[OS X]]), [[AmigaOS]], and [[IRIX]].[https://nmap.org/book/inst-other-platforms.html Other Platforms]. Nmap.org. Retrieved on 2013-02-01. Linux is the most popular platform, followed closely by Windows.name="Nmap Installation for Windows">{{cite web|url=https://nmap.org/install/inst-windows.html|title=Nmap Installation for Windows|publisher=nmap.org|accessdate=2008-05-14}}

==Features==
Nmap features include:
* Host discovery – Identifying hosts on a network. For example, listing the hosts that respond to [[Transmission Control Protocol|TCP]] and/or [[Internet Control Message Protocol|ICMP]] requests or have a particular port open.
* [[port scanner|Port scanning]] – Enumerating the open [[TCP and UDP port|ports]] on target hosts.
* Version detection – Interrogating network services on remote devices to determine application name and version number.name="vscan">[https://nmap.org/book/vscan.html Service and Application Version Detection]
* [[OS fingerprinting|OS detection]] – Determining the [[operating system]] and hardware characteristics of network devices.
* Scriptable interaction with the target – using [https://nmap.org/book/nse.html Nmap Scripting Engine] (NSE) and [[Lua (programming language)|Lua]] programming language.
Nmap can provide further information on targets, including reverse [[Domain Name System|DNS]] names, device types, and [[MAC address]]es.{{cite web |url=https://nmap.org/book/man.html |title=Chapter 15. Nmap Reference Guide |publisher=Nmap.org |date=2011-03-25 |accessdate=2011-04-23}}

Typical uses of Nmap:
* Auditing the security of a device or firewall by identifying the network connections which can be made to, or through it.[https://nmap.org/book/nmap-overview-and-demos.html Nmap Overview and Demonstration].
* Identifying open ports on a target host in preparation for auditing.[https://nmap.org/press/computerworld_when_good_scanners_go_bad.txt When Good Scanners Go Bad], From [http://www.computerworld.com/home/print.nsf/all/9903229872], ComputerWorld 22 March 1999
* Network inventory, [[network mapping]], maintenance and asset management.
* Auditing the security of a network by identifying new servers.[http://web.archive.org/web/20090401141545/http://heavyk.org/nmap-audit/nmap-audit-pod.html nmap-audit – Network auditing with Nmap]. heavyk.org
* Generating traffic to hosts on a network.[https://nmap.org/nping/ Nping - Network packet generation tool / ping utiliy].
* Find and [[Exploit (computer security)|exploit]] vulnerabilities in a network.{{cite web|last1=Leyden|first1=John|title=Revealed ... GCHQ's incredible hacking tool to sweep net for vulnerabilities: Nmap|url=http://www.theregister.co.uk/2014/08/15/gchq_port_scan_hacienda/|website=theregister.co.uk|publisher=The Register|accessdate=14 December 2014}}

==Graphical interfaces==
NmapFE, originally written by ''Zach Smith'', was Nmap's official [[GUI]] for Nmap versions 2.2 to 4.22.name="changelog" /> For Nmap 4.50 (originally in the 4.22SOC development series) NmapFE was replaced with Zenmap, a new official graphical user interface based on UMIT, developed by ''Adriano Monteiro Marques''.

Various web-based interfaces allow controlling Nmap or analysing Nmap results from a web browser.
These include LOCALSCAN,[https://web.archive.org/web/20080414002625/http://staff.washington.edu/dgreene/localscan/ ] archive copy on [[Archive.org]] nmap-web,[http://www.komar.org/pres/nmap-web/ nmap-web: quick-n-dirty web interface to Nmap]. Komar.org. Retrieved on 2011-09-17. and Nmap-CGI.[http://nmap-cgi.tuxfamily.org/ nmap-cgi homepage]. Nmap-cgi.tuxfamily.org. Retrieved on 2011-09-17.

[[Microsoft Windows]] specific GUIs exist, including NMapWin,[http://nmapwin.sourceforge.net/ NMapWin v1.2.3]. Nmapwin.sourceforge.net. ==Reporting results==
Nmap provides four possible input formats. All but the interactive output is saved to a file. Nmap output can be manipulated by text processing software, enabling the user to create customized reports.[https://nmap.org/book/man-output.html Output]. Nmap.org. Retrieved on 2011-12-10.

; Interactive: presented and updated real time when a user runs Nmap from the command line. Various options can be entered during the scan to facilitate monitoring.
; [[XML]]: a format that can be further processed by XML tools. It can be converted into a [[HTML]] report using [[XSLT]].
; Grepable: output that is tailored to line-oriented processing tools such as [[grep]], [[sed]] or [[awk]].
; Normal: the output as seen while running Nmap from the command line, but saved to a file.
; [[Script kiddie]]: meant to be an amusing way to format the interactive output [[l33t|replacing]] letters with their visually alike number representations. For example, ''Interesting ports'' becomes ''Int3rest1ng p0rtz''.

==History==
Nmap was first published in September 1997, as an article in [[Phrack Magazine]] with source-code included.[http://www.phrack.org/issues.html?issue=51&id=11#article Nmap Introduction – Phrack 51, Article 11]. Phrack.org. Retrieved on 2011-09-17. With help and contributions of the computer security community, development continued. Enhancements included operating system fingerprinting, service fingerprinting,name="vscan" /> code rewrites ([[C (programming language)|C]] to [[C++]]), additional scan types, protocol support (e.g. [[IPv6]], [[SCTP]][http://www.roe.ch/Nmap_SCTP SCTP Support for Nmap]. Roe.ch (2007-01-26). Retrieved on 2011-09-17.) and new programs that complement Nmap's core features. Changes include:
* 12 December 1998—Nmap 2.00 is released, including Operating System fingerprintingname="book-history">{{cite web |url=https://nmap.org/book/history-future.html#history |title=The History and Future of Nmap |publisher=Nmap.org}}
* 11 April 1999—NmapFE, a [[GTK+]] front end, is bundled with Nmapname="book-history" />
* 7 December 2000—Windows portname="changelog">{{cite web |url=https://nmap.org/changelog.html |title=Nmap Change Log |publisher=Nmap.org |accessdate=2011-09-17}}
* 28 August 2002—Rewrite from C to C++name="changelog" />
* 16 September 2003—The first public release to include service version detectionname="changelog" />
* 31 August 2004—Core scan engine rewritten for version 3.70. New engine is called ''ultra_scan''{{cite web |url=http://seclists.org/nmap-hackers/2004/10 |title=Nmap Hackers—Nmap 3.70 Released—Core Scan Engine Rewrite! |publisher=Seclists.org |accessdate=2011-09-17}}
* Summer 2005—Nmap selected for participation in [[Google Summer of Code]].{{cite web |url=http://seclists.org/nmap-hackers/2005/7 |title=Google sponsors Nmap summer student developers |publisher=Seclists.org |accessdate=2011-09-17}} Added features included Zenmap, NSE, Ncat, and 2nd-generation OS detection.
* 13 December 2007—Nmap 4.50, the 10th Anniversary Edition, was released. Included Zenmap, 2nd-generation OS detection, and the Nmap Scripting Engine{{cite web |url=http://insecure.org/stf/Nmap-4.50-Release.html |title=Nmap 4.50 Press Release |publisher=Insecure.org |accessdate=2011-09-17}}
* 30 March 2009—Emergency release of Nmap 4.85BETA5, leveraging NSE to detect [[Conficker]] infections{{cite web |url=http://seclists.org/nmap-dev/2009/q1/870 |title=Nmap Development—Nmap 4.85BETA5—Now with Conficker detection! |publisher=Seclists.org |accessdate=2011-09-17}}
* 16 July 2009—5.00 included [[netcat]]-replacement Ncat and Ndiff scan comparison tool{{cite web |url=https://nmap.org/5/ |title=Nmap 5.00 Release Notes |publisher=Nmap.org |accessdate=2011-09-17}}
* 28 January 2011—5.50 included Nping packet generation{{cite web |url=http://seclists.org/nmap-hackers/2011/0 |title=Nmap 5.50—Now with Gopher protocol support! |publisher=Seclists.org |accessdate=2011-09-17}}
* 21 May 2012—6.00 released with full IPv6 support.{{Citation needed|date=January 2014}}

The Nmap Changelog records all changes.name="changelog" />

==Ethical issues and legality==
Nmap is a tool that can be used to discover services running on Internet connected systems. Like any tool it could potentially be used for ''[[Black hat hacking|black hat]]'' hacking,{{cite web |url=http://www.securityfocus.com/news/10011 |title=Hacking tool reportedly draws FBI subpoenas |publisher=Securityfocus.com |date=2004-11-24 |accessdate=2011-09-17}} as a precursor to attempts to gain unauthorized access to computer systems. Nmap is more often {{citation needed|date=February 2015}} used by security and systems administration to assess networks for vulnerabilities.

System administrators can use Nmap to search for unauthorized servers, or for computers that do not conform to security standards.{{Cite web |url=http://www.techsupportalert.com/pdf/t04123.pdf |title=120 – How to conduct a security audit |publisher=Tech Support Alert |format=PDF |accessdate=2011-09-17}}

Nmap is related to vulnerability assessment tools such as [[Nessus (software)|Nessus]], which test for common vulnerabilities in open ports. The included NSE{{cite web |title=NSE scripts with brief summaries|url=https://nmap.org/book/nse-scripts.html|publisher=nmap.org |accessdate=2014-01-12}} scripts that are packaged with modern versions of Nmap are able to perform vulnerability checks against discovered services.

In some jurisdictions, unauthorized port scanning is illegal.{{cite web|title=First ruling by the Supreme Court of Finland on attempted break-in|year=2003|url=http://insecure.org/stf/fin.html|publisher=[[Osborne Clarke]]|accessdate=2010-02-21}}

==In academia==
Nmap is an integral part of academic activities. It has been used for research involving the [[TCP/IP]] protocol suite and networking in general.{{cite web |url=http://www.cs.utk.edu/~ltinnel/papers/200301IEEESecurityPrivacy_Validation.pdf | title=Validation of Sensor Alert Correlators}} As well as being a research tool, Nmap has become a research topic as well.{{cite web |url=http://www.springerlink.com/content/g37623k636656tu7 | title=A Data Mining Based Analysis of Nmap Operating System Fingerprint Database }}
